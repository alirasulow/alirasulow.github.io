        <h1>Непрерывная интеграция и непрерывное развертывание (CI/CD)</h1>
        <div class="date">21 марта 2026</div>
        
        <h2>Что такое CI/CD?</h2>
        <p>CI/CD — это методология автоматизации процессов разработки программного обеспечения.</p>
        
        <h2>Continuous Integration (CI) - Непрерывная интеграция</h2>
        <ul>
            <li>Разработчики часто сливают код в общий репозиторий</li>
            <li>Автоматическая сборка и тестирование</li>
            <li>Быстрое обнаружение ошибок</li>
        </ul>
        
        <h2>Continuous Delivery/Deployment (CD) - Непрерывная доставка/развертывание</h2>
        <ul>
            <li>Автоматическое развертывание на серверы</li>
            <li>Мгновенные обновления для пользователей</li>
            <li>Минимизация ручных операций</li>
        </ul>
        
        <h2>Популярные инструменты CI/CD</h2>
        <ul>
            <li><strong>GitHub Actions</strong> — встроенный в GitHub</li>
            <li><strong>GitLab CI/CD</strong> — для GitLab</li>
            <li><strong>Jenkins</strong> — открытая платформа</li>
        </ul>
        
        <h2>Пример GitHub Actions для GitHub Pages</h2>
        <pre><code>name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/configure-pages@v2
      - uses: actions/upload-pages-artifact@v1
      - uses: actions/deploy-pages@v1</code></pre>
